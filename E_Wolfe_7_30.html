Unit 7: Question 30: Talking about javascript dates.

a.

What does new Date()produce?

new Date() will give the following;

  • day of the week
  • the current month
  • the current day of the month
  • the current four digit year
  • the current time in hh:mm:ss format
  • and then the Time Zone.
You can then assign it to a variable and work with the data.

b.

What does new Date(2009, 0, 20) produce?

You get the same information as above but as applied to the date 01/20/2009. A more interesting question comes from the fact Javascript is using "0" for January.
What happens when you do new Date(2009, 0, 0)?

Then you have to ask why the inconsistency?

c.

If d1 and d2 refer to date objects, what does the expression d1.valueOf() - d2.ValueOf() produce?

Javascript dates are actually the number of milliseconds that have expired since 01 January, 1970 00:00:00 Universal Time (UTC).
One day contains 86,400,000 milliseconds, so we are dealing with large numbers.

Our expression will subtract the number of milliseconds contained in d2 from the number of milliseconds found in d1 and returm the difference.
